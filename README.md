# DummyJSON API Tests

Тестовое задание: API-проверки для публичного API DummyJSON с использованием pytest + requests.

## 📋 О проекте

Проект содержит набор API-тестов для проверки функциональности:

- Авторизация пользователей
- Получение данных пользователя
- Работа с корзинами (CRUD операции)
- Негативные сценарии

## 🚀 Запуск тестов

### Установка зависимостей

```bash
pip install -r requirements.txt 
```

### Запуск тестов

```bash
pytest -s -v
```

### Запуск тестов с генерацией Allure-отчёта

```bash
pytest --alluredir=allure-results
allure serve allure-results
```

### Запуск конкретного теста

```bash
pytest tests/test_auth.py -v
```

### Параллельный запуск тестов в pytest

```bash
pytest -s -v -n auto
```

## ✅ Реализованные проверки

### Авторизация (Auth)

1. Успешная авторизация (POST /auth/login) - проверка токена и структуры ответа
2. Неуспешная авторизация - неверный пароль, проверка 401

### Пользователь (User)

1. Получение текущего пользователя с токеном (GET /auth/me) - проверка данных
2. Получение пользователя без токена - проверка 401

### Корзины (Carts)

1. Получение корзин пользователя (GET /carts/user/{userId}) - проверка структуры
2. Получение корзины по ID (GET /carts/{cartId}) - проверка данных
3. Создание корзины (POST /carts/add) - проверка расчёта total
4. Обновление корзины (PUT /carts/{cartId}) - проверка модификации
5. Удаление корзины (DELETE /carts/{cartId}) - проверка isDeleted

### Негативные проверки

1. Несуществующий cartId - проверка 404
2. Невалидный body при создании корзины - проверка 400
3. Некорректное количество товара (0, отрицательное) - проверка валидации

## 📝 Примечания

Важно: DummyJSON API симулирует операции (POST, PUT, DELETE). Изменения не сохраняются на сервере, но API возвращает
корректные ответы с рассчитанными данными. Это учтено в тестах.
